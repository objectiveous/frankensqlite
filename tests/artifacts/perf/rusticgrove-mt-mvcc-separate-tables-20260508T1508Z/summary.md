# mt-mvcc Separate-Tables Artifact

## Purpose

Measure the concurrent-writer shape where each worker writes its own table
instead of sharing the same B-tree. This covers the README workload that was
previously called out as unmeasured.

## Command

```bash
/data/tmp/frankensqlite-rusticgrove-mt-separate-target/release-perf/mt-mvcc-bench \
  --separate-tables \
  --rows-per-thread=250 \
  --threads=1,2,4,8 \
  --iters=3 \
  --json-output=tests/artifacts/perf/rusticgrove-mt-mvcc-separate-tables-20260508T1508Z/separate-tables.json \
  --summary-md=tests/artifacts/perf/rusticgrove-mt-mvcc-separate-tables-20260508T1508Z/separate-tables.md \
  --history-json=tests/artifacts/perf/rusticgrove-mt-mvcc-separate-tables-20260508T1508Z/history.json
```

## Result

| Threads | Throughput ratio F/C | F p50 ms | C p50 ms | F failed | C failed |
|---------|---------------------:|---------:|---------:|---------:|---------:|
| 1 | 0.79x | 0.92 | 0.72 | 0 | 0 |
| 2 | 1.53x | 1.13 | 1.73 | 0 | 0 |
| 4 | 7.62x | 1.33 | 10.14 | 0 | 0 |
| 8 | 28.98x | 2.77 | 80.39 | 0 | 0 |

The 8-writer separate-table workload is no longer an unmeasured claim:
FrankenSQLite is faster than C SQLite WAL from 2 writers upward in this harness,
with the 8-writer row at 28.98x throughput and no failed rows on either engine.

## Files

- `separate-tables.json`: machine-readable benchmark report
- `separate-tables.md`: report generated by `mt-mvcc-bench`
- `history.json`: first pass-over-pass baseline for this workload shape
- `stdout/bench.out`: tabular stdout
- `stdout/bench.err`: command stderr and write confirmations
